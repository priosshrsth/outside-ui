# FormInput / `form.TextField`

A single-line text input with the standard label / description / error
chrome. Two entry points — pick the one that fits:

- **`<form.TextField name="email" label="Email" />`** — the one-liner.
  Registered on `useAppForm`, hides the TanStack Form render prop.
  Use this for 95% of forms.
- **`<FormInput label="Email" />`** — the underlying field component.
  Use it inside `<form.AppField>`'s render prop when you need to
  read field state for conditional rendering (showing a hint when
  the field is dirty, swapping icons based on value, etc.).

### Lazy-then-eager validation

<Canvas of={FormInputStories.Validation} />

Errors don't render until the user submits at least once. After that
first submit attempt, validation runs eagerly on change. Wire it by
attaching the same schema / validator to both `onSubmit` and `onChange`
on the field or the form — the display gate is built into the
component itself.

## Props

<Controls of={FormInputStories.Default} />

`form.TextField` accepts everything `FormInput` does, plus:

| Prop         | Type                  | Description                                                                                                                 |
| ------------ | --------------------- | --------------------------------------------------------------------------------------------------------------------------- |
| `name`       | `string`              | Dotted field path in the form values (e.g. `"user.email"`). Required.                                                       |
| `validators` | `FormFieldValidators` | TanStack Form field-level validators. Pass a Standard Schema (zod / valibot) or `{ onSubmit, onChange, onBlur }` functions. |

All native `<input>` props (`type`, `placeholder`, `autoComplete`,
`inputMode`, `maxLength`, …) are forwarded.

## CSS hooks

Style via these selectors; the library only ships structural CSS, so
you can skin freely without `!important`.

| Selector                                      | Element                                |
| --------------------------------------------- | -------------------------------------- |
| `[data-slot="form-field"]`                    | Outer grid container                   |
| `[data-slot="form-field"][data-layout="row"]` | Row-layout variant                     |
| `[data-slot="form-field"][data-invalid]`      | Container when error is displayed      |
| `[data-slot="form-field"][data-disabled]`     | Container when disabled                |
| `[data-slot="form-label"]`                    | `<label>`                              |
| `[data-slot="form-required"]`                 | Required-indicator `*` span            |
| `[data-slot="form-control"]`                  | Input box (border, radius, focus ring) |
| `[data-slot="form-input"]`                    | The `<input>` itself                   |
| `[data-slot="form-prepend"]`                  | Left affix slot                        |
| `[data-slot="form-append"]`                   | Right affix slot                       |
| `[data-slot="form-description"]`              | Helper text                            |
| `[data-slot="form-message"]`                  | Error alert                            |

## Accessibility

- Label and input are linked via `htmlFor` + `id` (ids are auto-generated).
- `aria-invalid` is set on the input when an error is displayed.
- `aria-describedby` points to the description and (when shown) the
  error message.
- `aria-required` mirrors the `required` prop.
- The error message has `role="alert"`, so screen readers announce it
  the first time it appears after a submit attempt.
- Prepend / append affixes are `aria-hidden="true"` — if the prefix
  carries meaning (e.g. a currency code), include it in the label or
  description instead.
